📊 VTM Trade Manager

🎯 Overview

The Veteran Trade Manager (VTM) is a sophisticated trade management system that combines market structure analysis with risk management to optimize trade exits. It uses hybrid logic that blends technical analysis (ATR, pivot points, resistance/support levels) with risk-reward ratios to place stops and take profits at optimal locations.

Core Features

  • ✅ Structure-based stop loss placement (wider, safer stops)
  • ✅ Hybrid take-profit calculation (market structure + R:R)
  • ✅ Partial profit taking with customizable sizing
  • ✅ Trailing stops for runner positions
  • ✅ Optional early profit locking
  • ✅ Asset-specific profiles (BTC, Gold)
  • ✅ Time-based exit monitoring

🛑 Stop Loss Calculation

VTM uses a sophisticated multi-step process to calculate stop losses that balance safety with giving trades enough room to work. The system prioritizes WIDER stops to avoid premature stop-outs from normal market noise.

The Calculation Process

  1. Calculate ATR (Average True Range)

    Measures recent market volatility using a 14-period ATR. This gives us the "normal" price movement range.

    atr = talib.ATR(high, low, close, timeperiod=14)
  2. Find Pivot Structure

    Identifies recent swing lows (for longs) or swing highs (for shorts) within the lookback period. This represents key support/resistance that price respected in the recent past.

    # For LONG: Find swing low where price[i] < price[i-1] and price[i] < price[i+1] # For SHORT: Find swing high where price[i] > price[i-1] and price[i] > price[i+1]
  3. Calculate ATR-Based Stop

    Places stop at a multiple of ATR away from entry (e.g., 2.0x ATR for BTC). Constrained to min/max % range.

    For LONG:
    atr_distance_pct = min(atr × multiplier / entry_price, max_stop_pct)
    atr_distance_pct = max(atr_distance_pct, min_stop_pct)
    atr_stop = entry_price × (1 - atr_distance_pct)
  4. Calculate Structure-Based Stop

    Places stop below the pivot point (for longs) with a full ATR buffer for safety.

    For LONG:
    structure_stop = pivot_low - (1.0 × atr)
  5. Choose the WIDER Stop

    Takes the maximum of ATR stop and structure stop, giving the trade more breathing room.

    Preliminary Stop:
    preliminary_stop = max(atr_stop, structure_stop)
    Key Insight: Using max() instead of min() is critical. We want the WIDER stop to avoid being stopped out by normal market noise. This is especially important for volatile assets like BTC.
  6. Apply Min/Max Constraints

    Ensures the final stop falls within the asset's safe range (e.g., 0.8%-1.5% for BTC).

    Final Stop:
    min_stop = entry_price × (1 - max_stop_pct)
    max_stop = entry_price × (1 - min_stop_pct)
    final_stop = max(min_stop, min(max_stop, preliminary_stop))
  7. Safety Check: Minimum ATR Distance

    For BTC, enforces a minimum of 1.5 ATR distance to prevent ultra-tight stops.

    If: |entry_price - stop| < (1.5 × atr)
    Then: stop = entry_price - (1.5 × atr)

📝 Example: BTC Long at $88,000

Given Data

Entry: $88,000 | ATR: $300 | Recent Pivot Low: $87,200

ATR Stop

$300 × 2.0 = $600 → $88,000 - $600 = $87,400 (0.68%)

Structure Stop

$87,200 - $300 = $86,900 (1.25%)

Chosen Stop

max($87,400, $86,900) = $87,400

Safety Check

Distance: $600 ≥ (1.5 × $300) = $450 ✓ OK

Final Stop Loss

$87,400 (-0.68%)

💰 Take Profit Calculation

VTM uses a hybrid approach that combines market structure (resistance/support levels) with traditional risk-reward ratios. This results in higher probability exits at natural turning points.

The Hybrid TP Process

  1. Detect Market Structure Levels

    Scans recent price history to find significant resistance (longs) or support (shorts) levels based on swing highs/lows that have been tested multiple times.

    # For LONG: Find swing highs above current price # Requirements: # - Price[i] > Price[i-1] and Price[i] > Price[i-2] # - Price[i] > Price[i+1] and Price[i] > Price[i+2] # - At least 2 touches within tolerance (1%)
  2. Calculate Risk-Reward Targets

    Calculates traditional R:R targets (1.5R, 2.5R, 3.5R) based on the stop distance.

    For LONG:
    risk = entry_price - stop_loss
    tp1 = entry_price + (risk × 1.5)
    tp2 = entry_price + (risk × 2.5)
    tp3 = entry_price + (risk × 3.5)
  3. Blend Structure with R:R

    For each target, checks if there's a structure level nearby. Uses structure if within 20% of R:R target and still meets minimum R:R requirement (1.2R).

    Decision Logic:
    1. Find closest structure level to R:R target
    2. Calculate: structure_rr = (structure_level - entry) / risk
    3. If |structure - rr_target| / rr_target < 20% AND structure_rr ≥ 1.2:
        → Use structure level
    4. Else:
        → Use R:R target
  4. Handle Missing Levels

    If structure levels are detected but they cap targets below intended R:R, the system may reduce the number of partials and redistribute position sizes accordingly.

📝 Example: BTC Long TP Calculation

Given Data

Entry: $88,000 | Stop: $87,400 | Risk: $600

R:R Targets (1.5R, 2.5R, 3.5R)

TP1: $88,900 | TP2: $89,500 | TP3: $90,100

Detected Structure

Resistance found at: $88,850, $89,600, $90,200

TP1 Decision

R:R target: $88,900 | Closest structure: $88,850
Difference: |88,850 - 88,900| / 88,900 = 0.06% < 20%
Structure R:R: (88,850 - 88,000) / 600 = 1.42R ≥ 1.2R ✓
→ Use structure: $88,850

TP2 Decision

R:R target: $89,500 | Closest structure: $89,600
Difference: 0.11% < 20% ✓
Structure R:R: 2.67R ≥ 1.2R ✓
→ Use structure: $89,600

TP3 Decision

R:R target: $90,100 | Closest structure: $90,200
Difference: 0.11% < 20% ✓
Structure R:R: 3.67R ≥ 1.2R ✓
→ Use structure: $90,200

Final Take Profits

TP1: $88,850 (1.42R) - Exit 33%
TP2: $89,600 (2.67R) - Exit 33%
TP3: $90,200 (3.67R) - Exit 34%

Why This Works: Market structure represents areas where price has historically struggled to break through. By placing targets AT these levels rather than blindly using R:R, you're exiting where the probability of reversal is highest. When no structure exists, R:R targets provide reliable fallback points.

📊 Position Management

Partial Profit Taking

VTM exits positions in stages to balance securing profits with letting winners run. The default split is three partials plus a runner.

Partial Target BTC Size Gold Size Action
TP1 1.5R (or structure) 33% 40% Lock stop at break-even after hit
TP2 2.5R (or structure) 33% 30% Activate runner trailing stop
TP3 3.5R (or structure) 34% 30% Final partial (if hit)
Runner Trailing stop Remainder Remainder Trails at 20% (BTC) or 12% (Gold)

Stop Loss Adjustment Logic

Initial Entry

Stop at structure-based level (e.g., -0.68% for BTC)

TP1 Hit (1.5R)

Exit first partial → Move stop to break-even (+0.1%)
Guarantees no loss on remaining 67%

TP2 Hit (2.5R)

Exit second partial → Activate trailing stop
Runner now trails at 20% (BTC) or 12% (Gold) from highest price

Runner Phase

Stop trails up automatically as new highs are made
Example: Price hits $90K → Stop moves to $72K (20% trail)

Early Profit Locking (Optional)

When enabled, VTM can lock in profits early to protect against reversals before first partial hits. This is particularly useful for Gold or overnight BTC swings.

# Enable with: enable_early_profit_lock=True early_lock_threshold_pct=0.01 # Lock at 1% gain
When to Use Early Lock:
• Gold: YES (1% threshold) - Tight moves, frequent reversals
• BTC Swings: YES (1.5% threshold) - Overnight holds, not monitoring
• BTC Position: NO - Need breathing room for big moves

💵 Position Sizing

VTM automatically calculates position size based on account risk percentage and stop distance.

Position Size Formula:

risk_amount = account_balance × account_risk (e.g., 1.5%)
trade_risk = |entry_price - stop_loss|
position_size = risk_amount / trade_risk

Risk: 1.5% = $1,500
